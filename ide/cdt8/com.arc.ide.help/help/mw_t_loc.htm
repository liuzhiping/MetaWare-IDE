Working with Local Variables

After your application is running in the debugger, you can display local variables in two different ways:

  • Select Window > Show View to open an Eclipse Variables view.
  • Select Debugger > Locals to open a debugger Local Variables window.

This topic tells how to use the debugger-supplied Local Variables window. For information about the Eclipse Variables view, see the Eclipse C/C++ Development User Guide.

4.14.2.1 Viewing Local Variables Currently in Scope

When you open the Local Variables window, all non-global variables currently in scope are displayed. All scopes from the current function's innermost scope out to the file scope are included. Non-exported (static) variables are also shown (non-exported variables are local to the current file).

Each line of text represents either the name and scope of a function, or one variable defined within the scope of that function. As the process executes, the debugger updates the Local Variables window to show the current value of each variable currently in scope.

All variables defined within a function's scope are listed below the line representing that function. In the case of nested scopes, the top-most scope displayed is the innermost one.

4.14.2.2 Changing the Display
  • To show or hide data types, click on the toolbar.
  • To show or hide addresses, click &.
  • To toggle alphabetical listing of variables, click .
4.14.2.3 Working with a Variable in the Local Variables Window

You can use the Local Variables window to select a variable, then perform the following debugging operations on the variable:

  • To add a variable to an Expressions window, select it and click .
  • Click to set a watchpoint on the variable.
  • Click to change the value of the variable.

To save the data to a file, right click and select Save to File.
