Searching Memory

To access the Memory Search dialog, select Debugger > Search memory when your application is running in the debugger (the Debugger menu is present in the menubar).

You can search the following ways:

  • Searching for an Integer Value
  • Searching for a Character String
  • Searching for a Hexadecimal String
4.13.2.1 Searching for an Integer Value

An integer can be an arbitrary expression that evaluates to a 1-, 2-, or 4-byte value. For example: 

r15
p->x
0x1234
a[i+1]

You can specify binary numbers with the syntax 0x2xbinary_digits. This is the same extension as in the MetaWare C/C++ language. For example, 0x2x1001_1110 is the same as 0x9e.

To search for an integer value, do the following:

  1. Select Debugger > Search memory to open the Memory Search dialog.
  2. Click the Integer value radio button if it is not already selected.
  3. Click the Integer size you want to search for.
  4. If you want to find the first occurrence that does not match the masked value you enter below, check Invert.
  5. Enter the integer value you want to search for in the Value: field, or choose a previously searched-for value from the drop-down list.
  6. You can search under a mask, where a match occurs if (search_int & mask) == (value_in_memory & mask). The default mask has no restrictive effect. If you want to specify a mask, enter your own hexadecimal value in the Mask: field.
  7. Enter the start address and the length of memory you wish to search.
    The debugger searches at the start address plus multiples of the size. For example, if you search for a two-byte integer starting at address 0x1000, the debugger searches locations 0x1000, 0x1002, 0x1004, and so on.
  8. Click a radio button to indicate whether you wish to search forward or backward (reverse) in memory.
  9. Click Search.

Results are reported in the Console view. For little-endian targets, the debugger extracts integers from the searched location with the appropriate endian reversal. For example, the value 0xaabb is represented in memory as the sequence of bytes bb aa. Thus searching for the integer 0xaabb is like searching for the hexadecimal string bbaa.

To search again starting from a search hit, click Search again.

4.13.2.2 Searching for a Character String

If you choose Char string, enter a sequence of characters with or without quotes. You can enter the wildcard character '?' to represent any single character, but not to represent zero or more characters.

You can also enter the following two-character escape sequences:

Represents This Character

Examples 
abc
"Hello, world.\n"
"Hello, ?????.\n"

To search for a character string, do the following:

  1. Select Debugger > Search memory to open the Memory Search dialog.
  2. Click the Char string radio button if it is not already selected.
  1. Enter the character string you want to search for between double quotes in the A char string field, or choose a previously searched-for string from the drop-down list. Note that the debugger does not provide a wildcard character '*' to represent "zero or more characters".
  2. Click a radio button to indicate whether you wish to search forward or backward (reverse) in memory.
  3. Click Search.

Results are reported in the Console view. To search again starting from a search hit, click Search again.

4.13.2.3 Searching for a Hexadecimal String

You can search memory for a hexadecimal string.

You can enter the wildcard character "?" for "any single hex digit," but not "*" for "zero or more digits."

Examples 
7ff
deadbeef
dea?beef

0x2x1001_1110 for 0x9e (MetaWare C extension: second 'x' for radix)

To search for a hexadecimal string, do the following:

  1. Select Debugger > Search memory to open the Memory Search dialog.
  2. Click the Hex string radio button if it is not already selected.
  3. Enter the hex string you want to search for between double quotes in the Hex string field, or choose a previously searched-for string from the drop-down list.
    A hexadecimal string can contain the wildcard character '?' denoting "any single hex digit," but the debugger does not support a wildcard character '*' meaning "zero or more hex digits."
  4. Select the address range you wish to search.
  5. Click a radio button to indicate whether you wish to search forward or backward (reverse) in memory.
  6. Click Search.

Results are reported in the Console view. To search again starting from a click Search again.
