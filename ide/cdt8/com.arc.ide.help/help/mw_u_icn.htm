Instruction-Count History Window

The Instruction Count History window shows disassembled code for your program. Labels of the form .0, .1, .2, and so on, are displayed in the left margin. Each label corresponds to a symbolic label and may be used to represent a symbol.

These labels change their associations when the Instruction Count History window is updated. You can use a label in a simple expression (for example, to specify the address of a breakpoint) for as long as the same values are on display -- after the Instruction Count History window is closed, all associations are discarded.

The margin includes indicators to show the following:

  • When the program has stopped on a line
  • When a stack frame is not currently active; the indicator shows where the program will be when control re-enters the stack frame
  • When a breakpoint is set on the line; when the letter 'T' is displayed, the breakpoint is specific to a thread, which you can identify using the Breakpoints window
  • When a breakpoint is disabled
