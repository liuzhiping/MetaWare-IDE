Configuring a CMPD Session

CMPD is the coordinated multiprocess debugger. To configure a CMPD session, do the following:

  1. With your project selected, select Debug Configurations... from the drop-down menu next to the Debug button.
  1. Select C/C++ Multiprocess Application, then click the New button:
The CMPD Debugger Configuration tab displays.
  1. Enter a name for your configuration in the Name: field.
  2. Click the Additional Settings tab to set command-line options, working directories, and other applications to launch. For more information, see Additional CMPD Settings.
  3. Check Start session with processes suspended if you want to control when processes launch manually instead of all of them launching at the beginning of the debug session.
  4. Click Add process to add a process for the session.

    Click OK to save this process and add and configure additional processes as desired. Clicking Edit allows you to configure each process individually. For more information, see Configuring Each Process Individually.
After you enter processes they appear in the Process table:

Here you can select processes and click the Edit or Delete buttons to edit or delete them.
  1. When you have all your processes configured, click one of the following:
    • Apply to save your entries and make additional settings on other tabs
    • Debug to start debugging
