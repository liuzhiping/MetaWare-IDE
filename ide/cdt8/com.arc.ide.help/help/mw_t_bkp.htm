Viewing and Managing Current Breakpoints

You can use the Breakpoints window to view breakpoints, enable and disable breakpoints, delete breakpoints, and view source code at a breakpoint location.

In This Topic
  • Viewing Breakpoints
  • Managing Breakpoints
4.9.2.1 Viewing Breakpoints

To see a list of current breakpoints, open a Breakpoints window by choosing Debugger > Breakpoints after your application is running in the debugger.

Each line in the listing shows the following information for a particular breakpoint:

  • 'H' to indicate a hardware breakpoint if applicable
  • the address of the instruction or source line
  • the name and line number of the source file (if available)
  • the text of the instruction or source line (if available)
  • whether the breakpoint is enabled or disabled
  • the attributes of the breakpoint, such as count or condition
If the debugger cannot find a source file associated with the breakpoint, or if the breakpoint is set on a machine instruction that exists outside of known source files, the Breakpoints window lists the breakpoint's address and symbolic label, and the disassembly code for the instruction.
4.9.2.2 Managing Breakpoints

You can manage breakpoints using the Breakpoints window toolbar.

You can use the buttons on the toolbar to do the following:

  • Click to remove the currently selected breakpoint
  • Click to remove all breakpoints shown
  • Click to enable or disable the breakpoint on the currently selected line
  • Click to enable or disable all breakpoints on the currently selected line
  • Click to create a new breakpoint (a Create breakpoint dialog opens)
  • Click to edit the attributes of an existing breakpoint (an Edit breakpoint dialog opens)
  • Click to open a Source window starting with the line where the breakpoint is set
