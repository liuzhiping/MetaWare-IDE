Adding Profiling by Right-Clicking in a Window

Depending on your target processor, you can right-click in some Debugger windows (such as Source and Disassembly) and select Profiling from the menu that appears,

You can then annotate the display lines with profiling address counters. Profiling choices vary according to the following:

  • Whether you are running your program on the simulator or hardware
  • Which other options you specified
  • Which target processor you specified

When you right-click in a window that allows profiling and choose Profiling, the following menu appears:

The upper section lists profiling data that you can view in a column on the left side of the current window. Select an item to display a column for any upper-section menu item.

The lower section lists profiling data that you can view in a separate window. For example, to view instruction counts in a separate window, select icnt display. Profiling windows show sorted displays of address counters, so you can see where your program has the highest count values.

You can enable profiling to gather statistics on instruction counts and killed instructions in the Target Selection area of the Debugger tab in the Debug Configurations dialog. For ARC 600 and ARC 700, you can enable hardware performance monitoring simulation in the same place.
