Building and Debugging

The MetaWare IDE includes various demo applications, including the one described here, which prints a series of prime numbers. Use the demo to familiarize yourself with the basic functioning of the MetaWare IDE and the MetaWare C/C++ toolset.

  • For more information on the tools, see the MetaWare Debugger User's Guide and MetaWare C/C++ Programmer's Guide.
  • For more demos, see the Project and Code Samples in the Welcome view (Help > Welcome).

The first part of the tutorial shows you how to create and build a project in the C/C++ perspective. Debug the Project shows you how to debug using the Debug Perspective.

2.1.1.1 Create a Demo Project

The MetaWare IDE workbench is organized into perspectives and views. A perspective is a collection of views or windows. The C/C++ perspective offers views for coding tasks, and the Debug perspective offers views for debugging tasks (other perspectives are also available; see Window > Open Perspective > Other...). The first word in the IDE title bar tells you which perspective you are in:

2.1.1.2 Introduce a Syntax Error and Rebuild
  1. If it is not already expanded, expand the prime item in the Project Explorer so that you can see prime.c.
  2. Double-click prime.c to open it in the Editor view.
  3. Introduce a syntax error by removing the semicolon at the end of the line 
    printf("Begin computation of Prime Numbers...\n")
  1. Save, then select Project > Build All to rebuild. The MetaWare IDE tries to rebuild the project and indicates the error in the Console and Problems views.
  2. Fix the syntax error by restoring the semicolon.
After you expand the Errors line, you can double-click the error in the Problems view to go directly to the source location where the error was detected (just after the missing semicolon in this case).
  1. Save the file, then select Project > Build All to rebuild.
You can have the IDE automatically rebuild projects when you save by selecting Build Automatically on the Project menu.
2.1.1.3 Debug the Project

To debug the prime project, do the following.

  1. Ensure that the prime project is selected by clicking the item in the Project Explorer.
  2. Click the Debug button to start debugging the project.

    The MetaWare IDE has a different set of views, called the Debug perspective, for debugging. A dialog appears to confirm that you want to switch to the Debug perspective. Check the box to remember your decision if you wish.
  3. Click Yes. The Debug perspective opens with prime.elf ready to debug.

    The debugger launches with a default configuration. The target processor is inherited from your C/C++ build settings. If you want to specify debugger options, including command-line options, click the down arrow next to the Debug button and select Debug Configurations.... This button is available in both the C/C++ and Debug perspectives.
The Debug Configurations dialog opens.
  1. Select the Debugger tab. Here you can set debugger options by clicking a line in the Debugger Options area. The command-line version of the currently selected options is shown in the Debugger Options: text box.
  2. Click some lines in the Debugger Options area and review the options you can select.
To set command-line options and toggles manually, click Command-Line Options in the Debugger Options area.
  1. Keep the default settings and click Debug to return to debugging.
Execution stops at an implicit breakpoint in main().

The application runs to completion and prints output in the Console view.
