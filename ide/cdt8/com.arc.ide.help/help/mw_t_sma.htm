Debugging with SmaRT Real-Time Trace (ARC 600 and ARC 700)

Select Debugger > SmaRT Trace to open the Instruction history -- SmaRT Trace window if your processor includes a SmaRT unit. Data are retrieved and displayed only when the processor is stopped.

NOTE
When you use the ARC 600 SmaRT module with the Automated Overlay Manager, it is possible but unlikely that the SmaRT trace might show invalid disassembly at the top of the display if one overlay triggers its eviction and replacement immediately by another overlay in the same memory area.

To enable or disable SmaRT capture, right-click in the Instruction history -- SmaRT Trace window select Enable SmaRT Capture on the popup menu.

By default, the window shows all instructions executed while trace capture is enabled, in order of execution, similar to the display in the Instruction history -- Simulator trace window, and has similar functionality; see Debugging with Instruction History for details.

Displaying Data for Flow Control Changes Only

To display only instructions involved in a flow-control change, right-click in the Instruction history -- Smart Trace window and select Flow-control Changes Only from the popup menu. Displaying only flow-control changes filters out sequences of more than one instruction between changes in flow control.

Handling of Innermost Loops

SmaRT Trace does not show every iteration of innermost loops; instead the SmaRT Trace window labels instructions within a loop as [repeated] to indicate that the preceding flow-control changes were repeated at least twice.
