Running a CMPD Session

After you configure a CMPD session, click Debug to launch it. The processes appear in the Debug view. Multiple instances of the same application are followed by a number in brackets [ ].

4.26.4.1 Basic Debugging

Select the process or processes to which you wish commands to apply.

You can apply any of the following to the selected process or processes:

  • Buttons on the Debug view toolbar (running and stepping commands, suspend, animation items, etc.)
  • Debugger commands you enter in the debugger Commands view
See the MetaWare Debugger User's Guide for special CMPD command-line commands.
When you set a breakpoint, the IDE attempts to apply it to every process if possible. The only way to limit a breakpoint to a single process is to set it using a command, for example:
[1] break queens.c!69
4.26.4.2 Additional Techniques
  • Debugger windows apply to the currently selected process. For example, the content of the Source window changes as you select different processes in the Debug view (unless the processes are suspended).
  • You can override the GUI and address processes by their [instance_number] and name using debugger commands. Thus, if process 1 is selected and you enter [2]run, the command applies to process 2, not 1.
  • Expand a process in the Debug view to see its threads. In a multithreaded application, you can double-click any thread in the expanded list to lock windows for that process to that thread.
  • To set a thread-specific breakpoint, do the following:
  • Set a breakpoint using the C/C++ Editor view.
  • Right-click your breakpoint and select Breakpoint Properties....
  • Click Filtering.
  • Check the threads to which to apply the breakpoint and click OK.
The ARC/MetaWare/arc/cmpd directory contains example applications that are discussed in detail in the MetaWare Debugger User's Guide.
