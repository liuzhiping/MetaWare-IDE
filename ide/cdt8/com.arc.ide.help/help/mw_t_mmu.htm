Debugging with ARC 700 Exception and MMU Registers

You can view the ARC 700 Memory Management Unit (MMU) and exception registers for hardware or simulator. To view the MMU and exceptions registers, first specify them in the in the Debug Configurations dialog, Debugger tab, Simulator Extensions > MMU/MPU Support before you start or restart debugging.

The MMU contents and Exception/MMU registers windows are available from the Debugger > Hardware menu after your application runs in the debugger far enough to activate the MMU. However, the MMU contents window is available only if you specified MMU in the Debug Configurations dialog (see Defining MetaWare Debugger Settings).

If your processor does not contain an MMU, the Exception/MMU registers window displays the exception registers and reports There is no MMU present. If you see this message when trying to view your MMU using a simulator, be sure you have specified an MMU in the Debug Configurations dialog, Debugger tab, Simulator Extensions > MMU/MPU Support.
