Downloading Additional Executables

Select Debugger > Download File... to load an additional executable when debugging an application that consists of multiple executables (same as the download filename command).

The IDE prompts you for the filename of the executable to download.

The debugger assumes the new executable was linked at locations that do not overlap the locations of the startup executable. The debugger does not check whether this condition is violated.

If you are debugging on hardware and additional executables are downloaded by other means, select Debugger > Load Symbols... instead. This method instructs the debugger to read the symbol table of the additional executables.
