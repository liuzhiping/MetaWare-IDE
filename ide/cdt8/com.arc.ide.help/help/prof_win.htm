Profiling Using the Profiling Window

The Profiling window lets you see and configure the execution profiling information provided by the MetaWare C/C++ run time and profile your application. Profiling employs an interval timer that interrupts the executing program with regular frequency. A counter is incremented whenever the program is interrupted. Since each counter covers a range of code addresses, the resulting table of counters gives an approximate idea of where the program is spending its time.

The Profiling window can also summarize how instruction count, cycle count, and cache-miss count are distributed among the functions in your program.

Some types of profiling data are displayed only if your program is compiled to provide them. If you receive a message that your program has no statistical profiling data or a profiling timer is not present, check your project's Properties > C/C++ Build dialog (or makefile, if yours is a standard-make project).

To view a profile of your application:

  1. Open the Profiling window by selecting Debugger > Profiling.
  2. Select the data to be summarized using the Address counters drop-down list. Some data is available only if you compiled with a timer.
  3. Click Get Profile.
    Statistical profiling data is not valid until your application has run long enough to gather statistics. A colored indicator shows when the statistical profiling data is valid:
  4. Make selections to modify any of the parameters. For detailed explanations of the parameters, see the MetaWare Debugger User's Guide.
Saving to a File

To save profiling output to a file:

  1. Right-click in the Profiling window.
  2. Select Save to File.
  3. Browse to the location for the saved file.
  4. Enter a filename and extension.
  5. Click Save.
