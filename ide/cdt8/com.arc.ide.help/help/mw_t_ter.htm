Specifying a Terminal or COMM-Port Simulator

The debugger provides terminal and COMM-port simulators:

  • You can use a terminal-window simulator for console I/O if you need to exercise an application that supports terminal I/O over a UART (such as an MQX application).
  • You can use a serial-port simulator for accessing actual hardware via a COMM port on your host computer (Windows only).

To specify the terminal or COMM-port simulator, to the following:

  1. Select Debug Configurations... from the drop-down menu next to the Debug button:
  2. Click the Debugger tab in the Debug Configurations dialog.
  3. Expand Simulator Extensions in the Debugger Options tree view and select Terminal/COMM simulator.
  4. Select the Terminal Simulator or COMM Port Simulator radio button.
  5. Set the parameters for your simulator in the resulting dialog or accept the defaults.
  6. Click Apply or Debug.

For information on the parameters you may set, see one of the following:

  • Debugger: Terminal Simulator Settings
  • Debugger: COMM-Port Simulator Settings
