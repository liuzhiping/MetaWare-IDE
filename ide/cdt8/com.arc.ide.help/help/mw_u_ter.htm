Debugger: Terminal Simulator Settings

This section lists the settings you may make in the Terminal Simulator area of the Debug Configurations dialog. For information about accessing the dialog, see Specifying a Terminal or COMM-Port Simulator.

Terminal Simulator Settings

If you select Terminal Simulator, use the check boxes and drop-down lists to specify the following for your terminal simulator:

  • Base address. Default is 0xfc1000. ARC VUARTs are located at 0xfc1000 + 256*N for the Nth device.
  • COMM Port number
  • TCP/IP Port for communicating with the terminal-simulator server.
  • Accept the default interrupt vector or select your own.
  • Select Use external terminal simulator to use the terminal simulator included with the debugger instead of the terminal simulator included with the IDE.
You might need to scroll the dialog to see additional settings.
  • Check Use default receive timeout to use the default timeout (200 ms) when waiting to receive a character, or select your own.
  • Check Use default send timeout to use the default timeout (200 ms) when waiting to send a character, or select your own.
  • (Optional) Log I/O. Select Write log file to log all outbound I/O. By default, the log goes to the terminal-simulator window (Windows hosts) or to stdout (UNIX and Linux hosts). You can enter a filename to send the log to a file or enter stdout to send the log to stdout (the default on UNIX and Linux hosts).
On Windows, check Omit generating window to send the I/O directly to the log file instead of generating a terminal-simulator window.
  • (Optional) Specify input file. Check Read input file and type or navigate to a file to read as input rather than the terminal window.

To specify more than one terminal simulator, specify the properties of each additional simulator on the command line using -simextp. You can use the -simextp=install_dir/bin/termide entry in the Debugger Options: field at the bottom of the dialog as an example.
