Debugger Conventions for Regular Expressions

Some debugger commands and some fields in debugger windows accept regular expressions as input. For example, the Name: and Goto: fields and the break command accept regular expressions as input.

To use a regular expression in an input field or as part of a command, follow the conventions listed below.

Convention
Matches
*
Zero or more of any character. For example, *.* matches all files.
?
Any one character. For example, f?nd matches find, fond, fund, and so on.
[ ]
Any one or more characters enclosed in square brackets. For example, [xyz] matches x, y, or z.
[^]
Any characters except the one(s) enclosed in square brackets. For example, [^xyz] matches any character except x, y, or z.
[-]
Any character within the specified range. For example, [a-z] matches any lowercase character from 'a' to 'z'.
