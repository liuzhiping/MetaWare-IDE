Creating a Build Configuration

The IDE provides build configurations appropriate for debug and release.You can create additional configurations to save build settings you use frequently. For example, you might want to switch easily to an alternate build with a different optimization level.

To create a new project configuration:

  1. Right click on the project and select Properties > C/C++ Build.
  2. Click the Manage configurations button in the Configuration: area.
The Manage Configurations... dialog appears (the title bar contains your project name, such as prime, followed by a colon and Manage Configurations).
Here you can add, delete, and select configurations for the project.
  1. Click New....
    The Create New Configuration dialog appears:
  2. Type a name for your configuration (for example, SmallCode) in the Name: box.
  3. Optionally enter a description in the Description: box.
  4. Select whether to Import from projects in your workspace (you can change the imported settings in Step Adjust the build settings for the new configuration in Properties > C/C++ Build > Settings.).
  5. Select whether to Import predefined initial configuration settings from a selection that is supplied (you can change the settings in Step Adjust the build settings for the new configuration in Properties > C/C++ Build > Settings.).
  6. Click OK to return to the Manage Configurations... dialog.
  7. Set your new configuration as the active configuration by selecting it in the Manage Configurations dialog and clicking Set Active.
  8. Note that the Status for the new configuration now reads Active, and click OK.
  9. Adjust the build settings for the new configuration in Properties > C/C++ Build > Settings.
  10. Click OK again to save the settings and return to editing.

After you rebuild your project with the new configuration set active, the new configuration's folder appears in the Project Explorer. Your generated executable is placed in the corresponding folder.
