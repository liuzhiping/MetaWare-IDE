Launching a Debug Configuration at Startup

By default, the IDE starts in the C/C++ perspective. You can save a step by invoking the IDE directly in the Debug perspective for a previously defined debug configuration. The debugger also starts, depending on your settings for Resume at startup and Stop on startup at: in the debug configuration's Debugger tab. Use the following command line: 

    mide.exe -launch configName

where configName is the name of a debug configuration that is defined in the current workspace.

Add -data workspace-path to specify an alternate workspace.

Note that the command line is case-sensitive. If the IDE cannot fine a debug configuration with the specified name, it displays a Command-line error box. After you dismiss the error box, the IDE starts as if the -launch option was not specified.
