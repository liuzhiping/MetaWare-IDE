Converting MetaDeveloper Projects

This topic describes how to import projects from the former MetaDeveloper product so that they work with the current IDE. This procedure works as long as the MetaDeveloper project space contains source files.

To Import a MetaDeveloper Project
  1. Select File > Import....
  2. Expand the Other node:
  3. Click Import old MetaDeveloper project space and click Next >.
    The Old MetaDeveloper Project-space Importer dialog opens:
  4. Type or browse to the location of your MetaDeveloper .psp file and click Next >.
  5. Change the project name, if desired, and choose one of the following:
    • Create new project in original folder creates a project in the same location as the old MetaDeveloper project using the old source files.
    • Create new project in workspace creates a new project in your workspace using the old source files.
    • Create new project in specified location creates a new project in the location you browse to or type in the text box, using the old source files.
  6. Select one of the following methods for source-code access:
    • Copy source files into new project location creates a copy of the files. Note that if you make any subsequent changes to the new source, the new source diverges from your old MetaDeveloper source.
    • Access original source files by link preserves a single copy of the source in its old location.
  7. Click Finish.

The IDE creates the new project using the source with a makefile based on your MetaDeveloper settings.
