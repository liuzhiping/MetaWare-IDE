Editing and Rebuilding in the Debug Perspective

The MetaWare IDE lets you edit your source in the Debug perspective. However, if you rebuild while debugging, the debugger continues debugging the old executable until you restart the executable, and debugger windows might lose synchronization.

After restart, the debugger engine detects that the file has changed and attempts to re-apply breakpoints at the same source location. If a source file has lines added or deleted, the breakpoints might no longer be in expected places.

  1. Ensure that the Debug perspective has prime.elf loaded and ready to debug as shown below (see Building and Debugging).
  2. In the Editor view for prime.c, locate the line that reads puts("\nEnd of Prime Number program.\n");
  3. Add (rebuilt in the Debug perspective) to the end of the text, so that the whole line now reads as follows: puts("\nEnd of Prime Number program (rebuilt in the Debug perspective).\n");
    An asterisk appears next to the filename prime.c in the Editor view to indicate that the file contains unsaved changes.
  4. Select File > Save then Project > Build All.
  5. Click the Restart button to inform the debugger that the application has changed:
  6. If execution stops at a breakpoint, click Resume:

    The prime.elf application runs to completion with the new output, which you can verify in the Console view:
  7. In the Editor view for prime.c, go back to the line that reads
    puts("\nEnd of Prime Number program (rebuilt in the Debug perspective).\n");
  8. Remove (rebuilt in the Debug perspective) to restore the original code:
    puts("\nEnd of Prime Number program).\n");
  9. An asterisk appears next to the filename prime.c in the Editor view. Select File > Save then Project > Build All to save and rebuild the file.
  10. Click the Restart button:
    When you run it to completion, the prime.elf now produces the original output, which you can verify in the Console view:
