Image Window

The Image window displays an image from data memory or other sources such as registers.

The image may be in one of several formats:

  • a sequence of bytes representing an 8-bit grayscale image
  • a sequence of halfwords representing a 16-bit grayscale image
  • a sequence of 32-bit words representing an ARGB (alpha-red-green-blue) image
  • a sequence of halfwords representing packed RGB (R5, G6, B5)
  • 4:2:0 YUV color
5.5.6.1 Configuring the Image Window

To specify the image display:

  1. Open the Image window by selecting Debugger > Image when your application is running in the debugger.
  2. Right-click in the Image window and select Configure > Configure image parameters.
  3. Specify parameters in the Configure image parameters dialog.
Reloading Image Parameters

After you specify the parameters and save them, you can reload them:

  1. Open the Image window by selecting Debugger > Image when your application is running in the debugger.
  2. Right-click in the Image window and select Configure > Reload prior parameters.
The image configuration applies to all Image windows.
5.5.6.2 Updating the Image Window

Using the Configure image parameters dialog you can specify whether the display updates the image every time execution stops. If you choose not to do this, such as to save memory traffic, you can click on the Update button to update the window at any time.

5.5.6.3 Adjusting the Image Display

An 8-bit grayscale image can be automatically "scaled," i.e., the range of values found in the image can be expanded to fill the entire 0...255 range. The 16-bit grayscale image is already scaled to 0...255.

The image can be flipped and rotated and zoomed by clicking the Flip and Rotate buttons.

Zooming might not work on some versions of Windows. The symptom is that the image disappears for all zoom settings except 1:1.
