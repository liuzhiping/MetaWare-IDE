Viewing All Source Files

When your application is running in the Debug perspective, select Debugger > Source files for an alphabetical listing of all the application's source files for which the debugger has line information.

The Source files window also lists the names of all functions and/or file-scope variables that are declared in a source file: clicking the '+' in the left margin expands the list.

  • To restrict the files in the list, enter a regular expression such as *.h, *.cc, or *input* in the Reg expn: text box and press Enter.
  • To view the source code for a file, function, or variable, select a a line and click .
  • To toggle alphabetic sorting of the list, click .
  • To show or hide functions, click .
  • To show or hide variables, click .
  • To show or hide data types, click .

Double-click a line to do the following:

  • If the line shows a source file, a Source window opens for that file.
  • If the line shows a variable, the variable is shown in an Examine window.
  • If the line shows a function, a breakpoint is set at the function entry point.

To view a list of all currently loaded software modules (regardless of their source files), select Debugger > Modules. The resulting display includes any DLLs (dynamic link libraries) or SOs (shared objects on UNIX), with an arrow to indicate the currently active module.
