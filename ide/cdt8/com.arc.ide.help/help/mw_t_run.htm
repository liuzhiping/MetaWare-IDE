Defining MetaWare Debugger Settings

In the IDE, you use a debug configuration to launch an executable in the debugger. The debug configuration contains all the debugger settings for your executable.

To define MetaWare debugger settings, you create a debug configuration:

  1. In the C/C++ perspective, select a project in the Project Explorer pane.
  1. Open the drop-down menu next to the Debug button and select Debug Configurations....
The Debug Configurations dialog opens. Here you can create, manage, and launch debug configurations.
  1. In the Name: field, specify a for your debug configuration or accept the default.
  2. Click the Debugger tab.
  1. Set debugger options by selecting an item from the tree in the Debugger Options area.
The command-line version of the current options is shown in the Debugger Options: text box. To set command-line options and toggles manually, select Command-Line Options in the Debugger Options tree.
Under Target Selection, the Select the target: drop-down lets you select debugging on hardware and various simulators (not all options are available for all target processors).
          • MetaWare ARC Simulator
          • Synopsys Virtualizer
          • Hardware
          • nSIM
          • SystemC
          • xCAM or RTL simulation
See also Debugging with Special Simulators and Using a Digilent Cable to Connect to Hardware.
  1. Leave Resume at startup checked to have the debugger implicitly start running the application, or uncheck Resume at startup to stop at the entry-point address (typically _start).
  2. Check Stop on startup at: to have the debugger set an initial temporary breakpoint at the address of the function specified (default is main()) or another function you name. The initial resume is expected to stop there.
  3. Click Debug to switch to the Debug perspective and start debugging.
