Coordinated Multiprocess Debugger (CMPD)

The Coordinated Multiprocess Debugger (CMPD) extensions to the MetaWare debugger let you debug multiple processes at the same time.

CMPD extensions let you perform tasks such as the following:

  • Debug multiple processes, applying commands to one, some, or all
  • Debug multiple copies of the same process on different processors and operating systems
  • Debug multiple processes running on different processors and operating systems
  • Run processes on the appropriate processor in a multicore SoC to debug the whole SoC simultaneously

In a CMPD session, a command applies to all processes or only to a subset that you select.

How It Works

The debugger creates a command processor to execute toolbar actions or command-line commands for each process. The command processors remain as processes come and go. A process might complete, and you might start debugging another process; CMPD debugs the new process using the same command processor as the process that just completed. If you restart a process, the debugger kills the process and reloads it as a distinct process, but again using the same command processor.

You select the process or processes to apply commands to in the Debug view.
