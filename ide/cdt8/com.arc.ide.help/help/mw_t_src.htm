Viewing Source

If your program contains debug information, you can view the source two ways in the Debug perspective:

  • In the Eclipse C/C++ Editor view
  • In the debugger Source window

Both displays indicate your current location as you run or step your program, and both displays let you set and remove breakpoints.

  • The C/C++ Editor lets you edit your source, save, and rebuild without leaving the Debug perspective. For more information about the C/C++ Editor view, see the Eclipse C/C++ Development User Guide.
If you rebuild while debugging, the debugger continues debugging the old executable until you restart the executable, and Debugger windows might lose synchronization. After restart, the debugger engine detects that the file has changed and attempts to re-apply breakpoints at the same source location. If a source file has lines added or deleted, the breakpoints might no longer be in expected places.
  • The Source window permits source lines to be annotated with profiling information by right-clicking and making choices from the resulting pop-up menu.

To open a Source window, select Debugger > Source after your program is running in the debugger.

4.7.1.1 Working with Breakpoints in the Source Window

To set or remove a breakpoint, do any one of the following in the Source window:

  • Double-click a line of source.
  • Right-click a line of source, and then click Break.
  • Select a line of source, and then click the Break button:
4.7.1.2 Profiling in the Source Window

If the code was compiled with profiling information, you can view profiling data in the Source window.

  1. Right-click, and then select Profiling.
  2. Select any number of menu items from the top half of the menu.
A column is added to the Source window that contains a profile for the type of profiling you chose.
Reselecting the item removes the column.
4.7.1.3 Troubleshooting the Source Window
Problem: Source window is not showing source code
If the Source window displays Not stopped in known source, your source file may have been moved, renamed, or deleted from the directory in which it was compiled.
Solution
Select Debugger > Command-line Input and type: set source_path=path, where path is the full path to the source file.
Example 
   set source_path=c:\arc\metaware\arc\demos\
Problem: Unable to set a breakpoint
To set a breakpoint, you must first compile with debugging information (-g).
