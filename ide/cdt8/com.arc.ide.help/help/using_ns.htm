Using nSIM

The nSIM simulator is significantly faster than the MetaWare ISS, although it does not have all the capabilities of the MetaWare ISS; see the nSIM documentation for details.

If you are using the nSIM Synopsys Virtualizer nSIM SystemC implementation, you do not have to set up a SCIT connection if you select Synopsys Virtualizer instead of nSIM as the target.

You can configure nSIM using the Debugger Options tree view:

  1. In the C/C++ Projects view, select a project.
  2. Open the drop-down menu next to the Debug button and select Debug Configurations....
  3. Click the Debugger tab.
  4. Ensure that MetaWare Debugger is selected in the drop-down menu.
  5. In the Debugger Options tree view, select Target Selection.
  6. Select nSIM from the Select the target drop-down list:
The target processor is inherited from the build configuration. See Defining MetaWare Build Settings.
  • Select Enable JIT (just-in-time compilation) for maximum performance. You can further specify the number of JIT threads to use (improves performance on multicore hosts).
  • Select SystemC/SCIT Simulation to connect to an nSIM session already started using SystemC or a SCIT server. For more information, see the nSIM documentation for more information.
  • Enable Instruction History Tracing must be selected to enable the Instruction history -- Simulator trace window (see Debugging with Instruction History). Note that this option increases performance overhead.
  • Click OK after making your selections.
  • Click Debug to start debugging.
