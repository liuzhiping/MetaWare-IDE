Debugging with Overlays (ARCompact and ARCv2)

The MetaWare debugger automatically understands applications that use the Automated Overlay Manager and takes care of the switching behind the scenes so that you can debug normally.

When you use the ARC 600 SmaRT real-time-trace module with the Automated Overlay Manager, it is possible but unlikely that the SmaRT trace might show invalid disassembly at the top of the display if one overlay triggers its eviction and replacement immediately by another overlay in the same memory area.
4.19.0.1 Viewing Overlays

To open the Overlays window, select Debugger > Overlays. The menu item is not available if the debugger does not detect overlays in the application.

The Overlays window is an XML tree view that displays the overlay executable name and memory-region size (as hex and decimal values), with nodes for each overlay set in the application.

Some items appear only for currently loaded overlays.
Changing Sort Order
  • To toggle sorting of sets by number or address, click Sets sorted by set number or Sets sorted by address.
  • To toggle sorting of the overlay mapping table (OMT) by index or address, click OMT entries sorted by index or OMT entries sorted by address.
