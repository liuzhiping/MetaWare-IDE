Opening a Disassembly Window

To open a Disassembly window, launch your debug configuration and select one of the following:

  • Debugger > Disassembly (debugger Disassembly window)
  • Window > Show View > Disassembly (Eclipse Disassembly view)

The Eclipse Disassembly view is documented in the Eclipse C/C++ Development User Guide; this section describes the debugger Disassembly window.

Each line of text represents a single disassembled machine instruction. As you step through your program, an arrow points to the current location in disassembly code. The disassembly code is displayed in order by address, starting at the current location of the program counter. You can scroll up to view previously executed instructions.

Indicators in the left margin show the status of the program, stack frame, and breakpoints. Note the visual difference between enabled and disabled breakpoints.

Labels of the form .0, .1, .2, ... are displayed on the left side of the listing. Each label corresponds to a symbolic label and may be used to represent a symbol. The labels change their associations when the Disassembly window is updated. You can use a label in a simple expression (for example, to specify the address of a breakpoint) for as long as the same values are in the window. After you close the Disassembly window, all associations are discarded.

You can open more than one Disassembly window.
