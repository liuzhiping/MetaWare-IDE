Examining Variables

Use an Examine window to inspect the value of a variable or the value of an element of an aggregate data structure (array element, structure member, and so on). You can also toggle on and off the display of the variable's data type.

The Examine window is available only for applications compiled with debugging information (option -g).

To open an Examine window, select Debugger > Examine while your application is running in the debugger, or right-click and select Examine in a Global Variables or Local Variables window.

The Examine window opens.

You can examine items by typing them in the Examine: input field and pressing Enter. To remove an item, right-click it and choose Remove from the pop-up menu that appears.

  • To show or hide data types, click (...) on the toolbar.
  • To show or hide addresses, click &.
  • Following an item shows the component parts of an aggregate or dereferences a pointer. To follow an item, right-click it and select Follow, or select the item and then click the Follow button.

You can also select a line and do one of the following:

  • Click to set a watchpoint on a variable.
  • Click to change the value of a variable.

To save the data to a file, right click and select Save to File.

4.14.3.1 Examining Deeper Levels in an Aggregate Data Structure

When you open an Examine window on an aggregate data structure, the Examine window's default configuration is tree-structured; that is, all components of the data structure are listed below it, joined by a tree diagram. Components that are themselves data structures are marked with a data-structure symbol.

Generally, you can learn all you need to about a simple data element by looking at it in the original Examine window. There are several ways you can examine the components of a data structure that are themselves nested structures.

  • Click the nested-data symbol ([+]) next to the structure to expand the tree structure and display the structure's elements in the current Examine window.
You can navigate through many levels of nested structures in this way.
  • Click the structure to select it, then click Follow on the Examine Options toolbar to display the structure by itself in the current Examine window.
This method is useful if you have a very large data structure or one that contains nested components with many levels; examining all the levels of such a structure at once in an Examine window can result in a very large, awkward tree structure.
4.14.3.2 Reevaluating the Value of Data Structures

You can use the Deep/Shallow Binding button to switch between deep or shallow binding. In deep binding (the default), the address is computed just once. In shallow binding, the expression containing the data structure is re-evaluated each time the Examine window is refreshed (typically every time execution stops).

To reevaluate a data structure, click the button to its left.
