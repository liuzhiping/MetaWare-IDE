Using the Cycle-Estimating Simulator

The ARC Cycle-Estimating Simulator (CES) is a component of the ARC ISS that estimates machine cycles in executing programs. The CES also includes a cache thrash analysis routine to point out cache thrashing.

CES works with both instruction and data caches. Cycle estimation is not available for code fetch where no icache is present.

To enable cycle estimation, select Enable estimated instruction cycle counting in Simulator extensions > Cycle Counting in the Debugger tab tree view when you create your launch configuration for debugging.

To analyze performance of icache or dcache:

  1. Before launching your Debug configuration, select Simulator extensions > Cache simulation and check Instruction cache or Data cache (or both) in the Debugger tab of the Debug Configurations dialog.
  2. Debug your application. The debugger adds additional cycle-estimating registers in the Registers window.
  3. Select Debugger > Instruction cache or Debugger > Data cache to open an Instruction cache or Data cache window.
  4. Click the Analyze button to open an Instruction cache analysis or Data cache analysis window with cache analysis for the selected cache.

The Data cache analysis or Instruction cache analysis window opens.

  • To restrict the display, enter a regular expression such as Set 02* in the Reg expn input field, or choose a previous entry from the drop-down list, and press Enter.
  • To show or hide line numbers, click the button.
