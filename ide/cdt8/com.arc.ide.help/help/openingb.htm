Opening an Instruction history - Simulator trace Window

To open an Instruction history - Simulator trace window, launch your debug configuration and select Debugger > Instruction history.

The window includes indicators to show the following:

  • when execution is stopped at the location specified by a breakpoint.
  • when a breakpoint exists at a location.
  • when the breakpoint at a location is disabled.
  • when the line that is being shown in the assembly display.
  • when the location has a marker on it.
  • for multi-threaded programming only, an indicator containing the letter 'T' within it means the breakpoint is specific to a thread. You can determine which thread by looking at the breakpoint in the Breakpoint display.

The Instruction history - Simulator trace window displays the result of a trace capture of the executing program. Typically the trace capture comprises the instructions most recently executed by the program. The instructions are retrieved from a trace buffer maintained by the target.

The instructions are listed with the most recent instruction at the bottom of the display. The instruction numbered 0 is the last instruction executed.

4.17.1.1 Saving to a File

To save the window contents to a file:

  1. Right-click within the window and select Save to File from the popup menu.
  2. Save to the file and directory of your choice in the resulting dialog.
