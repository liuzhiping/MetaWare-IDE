Working with the Debugger Memory Window

To view the contents of a region of memory associated with your program, use the Memory window. By default, the Memory window shows the contents of memory in ascending order by address.

To open a Memory window, select Debugger > Memory.

Each Memory window includes a number (such as #1) that you can use to compare several Memory windows.

By default, the Memory window shows 16 bytes of memory per line, grouped in hexadecimal, four-byte words, in ascending order from the starting address you specified.

Each line of text in the Memory window begins with an address on the left (for example, 0804826c), followed by a string of 32 hexadecimal digits representing the contents of 16 bytes of memory starting at that address.

Changing Display Parameters

You can use the window controls to change the display:

  • Starting point. Enter the register name, hexadecimal start address, or symbolic label of a memory region from which to begin the display, then press Enter.
  • Base. Choose the numeric base in which to display the memory values.
  • Bytes per line
  • Size. Choose the data type in which to display the memory values.
  • Change the value at a memory location (select the line first).
  • Toggle display of columns by increasing or decreasing offsets (addresses).
  • Refresh the display
Changing a Value

To change a value in memory:

  1. Click a line to select it.
  2. Click the Change button or right-click and select Change.
  3. Enter the new value and click OK.
Examining a Location

To examine a memory location in an Examine window, right-click a line, and then select Examine from the pop-up menu.

The selected address is displayed with the prefix *(int*), which means that the address has been cast to a pointer-to-integer. You can then change the int part of the prefix to any type. Subsequent examine selections from the Memory window change the address only: Your selected type preference is preserved.

The Eclipse the Memory display available from Window > Show View provides essentially the same functionality.
