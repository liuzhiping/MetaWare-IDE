Viewing ARC Enhanced System Timers

If you build your application with an enhanced system timer, the debugger automatically detects the timer when you launch your debug configuration.

To build your application with an enhanced system timer, specify one in your project's Properties > Settings > Tool settings > Processor/Extensions.

If you specify a timer when you build a project, the timer is unconditionally enabled in the debugger. If you don't specify a timer when you build a project, then it may be enabled or disabled in the debugger (i.e., timers previously specified in a project's build settings might continue to display if unspecified in later builds. You can disable them in the Debug Configurations dialog, Debugger tab, Simulator Extensions > Timers).

4.18.4.1 Opening a Timer Window

To open a window for your timer when debugging, select Debugger > Hardware > TimerN (where N is 1 or 0).

If you build with a timer but it does not appear on the menu, you can specify it in the Debug Configurations dialog, Debugger tab, Simulator Extensions tree item.

The timer windows show the contents of each bit in a tree view, like the windows for extension registers.
