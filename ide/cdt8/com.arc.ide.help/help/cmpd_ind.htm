Configuring Each Process Individually

You can use the Edit Process dialog to specify the following:

  • Process Name: Your name for the process must start with an alphabetical character and may not include spaces.
  • Instance Count: The number of instances of this process you want to debug
    Select Project: The project where the process resides
  • Path and arguments for the process
    Note: If the path contains spaces, it must be quoted. Backslashes must be escaped as double backslashes (e.g., "C:\\Document and Settings\\..."). If you use the Search Project... or Browse... buttons, the IDE handles any necessary quoting and escaping automatically.
  • Debugger settings
4.26.3.1 Configuring Debugger Settings for Each Process
  1. Click Configure... to configure debugger settings for each process.

    A Configure process window opens with the same tree view as in the Debugger tab in the Debug Configurations dialog. Note that if you select Program Options, the dialog allows you to specify the RTOS for the current process.
  2. Make any settings you desire in the dialogs associated with each item in the tree view and click OK.
  3. Click OK to save this process and add and configure additional processes as desired.
    After you enter processes they appear in the Process table:
    Here you can select processes and click the Edit or Delete buttons to edit or delete them.
  4. Use the Additional Settings tab to select directories, if desired:
  5. The default Working directory for target applications is the directory where they are launched. To specify a working directory, uncheck Use default and enter a new directory, or use the buttons to specify one.
    • Workspace... opens a Folder Selection dialog so you can select a working directory in your workspace.
    • File System... opens a Browse for Folder dialog so you can browse and select a working directory anywhere on your file system.
    • Variables... opens a Select Variable dialog where you can select Eclipse directory variables related to your project. Enter a string in the Choose a variable box to filter the listed variables. You can specify a subdirectory as an Argument to the variable. Click Edit variables... to open a Preferences dialog where you can create New, Edit, or Remove variables.
  6. Check Application to invoke prior to launch: to specify an application to invoke before invoking the debugger, such as a SCIT server process that a debugger DLL might need to communicate with.
You can specify a Command to invoke prior to debugger launch and the Working directory from which to launch. If you need more time before the debugger starts, you can specify the number of seconds to Delay before launching the debugger.
  1. When you have all your processes configured, click Apply to make additional settings or Debug to start debugging.
