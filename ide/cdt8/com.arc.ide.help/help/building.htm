Building Projects from the Command Line

After you define build settings for your projects, you can build one or more projects from the command line without launching the IDE. This can be useful for automated nightly builds, for example.

If errors occur, a dialog opens and must to be dismissed before the build continues. For automated builds, include Eclipse option --launcher.suppressErrors.

When you specify -build or -cleanBuild, the IDE is not invoked. Instead, the specified projects are built, and then the command terminates. The build process runs silently unless you specify -console.

You can also build a specific configuration of a project by including it in the command line after the project. For example, mide -data myWorkspace -build myProject/Debug builds only the Debug configuration for the project named myProject.

On windows, the variable %ERRORLEVEL% is set to 1 in the case of an error and 0 if the build is clean.

4.2.4.1 Command-Line Options

Type mide, followed by one or more of the following command-line options.

-build {project_name | all}

Build all configurations of project project_name.

  • Specify this option multiple times to build more than one project.
  • Use -build all to build all projects.
-cleanBuild {project_name | all}

Clean, then build all configurations of project project_name.

  • Specify this option multiple times to clean and build more than one project.
  • Use -build all to clean and build all projects.
-console

Launch a console window to view build output. If this option is not specified, the build process runs silently.

-data workspace_path

Specify the path to the workspace.

-import [uri:/]/path/to/project

Specify the path to a project to be imported into the workspace prior to the build. This option is only required if the referenced project is not already in the workspace. You can specify this option multiple times to import more than one project.

4.2.4.2 Examples 
//Clean and build projects in a specific workspace; display output on 
console.
mide -data C:/workspace -cleanBuild TestProject -cleanBuild MainProject 
-console

//Build a project in a specific workspace with console display and no error 
dialog.
mide -build Queens -data C:\ARC\DemoWorkspace -console 
--launcher.suppressErrors

//Build a project in a specific workspace and send errors to a file.
mide -build Queens -data C:\ARC\DemoWorkspace --launcher.suppressErrors > 
errors.txt
4.2.4.3 Improving Build Speed

Disabling the automatic C/C++ indexer can improve build speed.

  • To disable the C/C++ indexer for the entire IDE, uncheck Window > Preferences > C/C++ Indexer.
  • To disable the C/C++ indexer for a single project:, go to Project properties > C/C++ General > Enable project specific settings.
