Lightweight Events Window

The LIghtweight Events window displays the state of lightweight events.

Displayed Fields

LWEvent - the address (or variable name) of the event descriptor.

Valid - indication as to whether or not the descriptor is valid. This field reads No if the event descriptor appears to be corrupted.

Value or BitMask - the current settings of this event group (i.e., which of the 32 individual events are set or cleared)

AutoClear - Indicates whether autoclear is set for this event. Events can be created with a property to autoclear themselves, so that after an event is set, a task is readied, then the event is automatically cleared, so the application does not have to do it.

Waiting# - number of tasks that are suspended waiting for this event.

The expanded tree contains a list of tasks waiting for the event. Each task can also be expanded to see its description.

Waiting Task Field Details

Waiting Task - a task that is waiting for this event.

All/Any - Indicates whether to wait until all bits in the bitmask are set before returning from the wait. If this property is set to Any, wait only until any of the bits is set (similar to a logical AND vs OR for waiting).

Waiting BitMask - Indicates the group of bits being waited for. Can be a single event bit, or a combination. The All/Any property helps determine what to do when one or more of these bits is set.
