Semaphores Window

The Semaphores window lists the semaphores. Each entry contains the current state of the semaphore, the semaphore's attributes, the tasks waiting for the semaphore, and the tasks that own the semaphore.

Displayed Fields

Sem

The address or variable name of the semaphore descriptor.

Owning#

The number of tasks that own the semaphore.

Waiting#

The number of tasks waiting for the semaphore

Destroy

Indicates if the semaphore is in a delayed destroy state.

Count

Number of available connections to the semaphore. When a task waits for a semaphore, MQX decrements Count and connects the task to the semaphore. When Count reaches 0, tasks that attempt to wait for the semaphore are suspended until the semaphore becomes available. When a task posts or closes a semaphore connection, MQX increments Count.

Name

Identifier that was given to the semaphore.

Additional Semaphore Details

You can see additional details about a semaphore by expanding its entry in the table:

Max Count

Maximum number of semaphore connections that can be available at any time. The task that creates the semaphore sets the value.

Priority Q'ing?

If Yes, then the highest priority task obtains the semaphore. If No, then the first task to request the semaphore obtains it.

Priority Inherit?

Priority inheritance might affect the priority of tasks that own the semaphore. If Priority Inherit? is Yes and if a task waiting for the semaphore is of higher priority than any of the owning tasks, MQX boosts the priority of the lower-priority owning tasks to the priority level of the waiting task. When the owning tasks release the semaphore, MQX returns their priority to the original level.

If Priority Inherit? is No, MQX takes no action.

Task List

This list contains an entry for each task that owns the semaphore and an entry for each task that is waiting for the semaphore.
