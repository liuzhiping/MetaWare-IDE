User Logs

User logs are also listed in the Log Summary window.

Displayed Fields

Log Component

Indicates whether the log component is installed. Logging can occur only if the component is installed.

Log Enabled

Indicates whether log write operations are enabled. At any time, an application can disable a log (no write operations are allowed) by calling _log_disable(). The application can subsequently enable the log by calling _log_enable().

Log Used

The format of the field is current log size/maximum size. When an application creates a log, it sets the maximum size, which is the maximum amount of data (in long words) that can be written into the log. Current log size is the amount of data written into the log so far.

Overwrite

If this field is set to Yes, then new entries overwrite the oldest entries when the log becomes full. If it is set to No, then no further write operations occur when the log becomes full.

Log Entries

Each log entry is represented by an entry in the list. Each entry contains a sequence number, a timestamp, and application-defined data. The log entries are displayed in chronological order, with the oldest entry first. If the first entry is not sequence number 1, the oldest entries in the log were overwritten. MQX assigns the timestamp when it logs the entry.
