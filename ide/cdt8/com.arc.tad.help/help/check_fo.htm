Check For Errors Window

The Check For Errors window provides a list of possible problem areas that were detected when the MQX internal data structures were examined for correctness.

Displayed Fields

All internal MQX data structures are examined and any detected data corruption, system errors, or application errors are indicated. The types of errors that are monitored include the following:

  • tasks with a non-zero task error code
  • tasks with a stack overflow
  • interrupt stack overflow
  • highest memory address used and end-of-pool value
  • memory blocks that fail a checksum verification
  • maximum allowed message-queue number
  • message queues that are full (that is, the number of entries is equal to the maximum allowed)
  • messages in a message queue whose header fails a validity check
  • partitions that do not pass a validity check
  • event structures that fail a validity check (structures include the event component structure, all event structures, all name structures for event, and all event connection structures for all event groups)
  • semaphore structures that fail a validity check (structures include the semaphore component structure, all semaphore structures, all name structures for semaphores, and all semaphore-connection structures for all semaphores)
  • current number of message pools and maximum number of allowed message pools
  • message pools with an invalid header
  • messages with an invalid header in a message pool
  • task queues with an invalid header
  • lightweight semaphores with an invalid header
  • mutex structures that fail a validity check (structures include the mutex component structure and all mutex structures)
  • tasks that are blocked because of an unhandled interrupt
