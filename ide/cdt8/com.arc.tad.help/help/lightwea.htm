Lightweight Timers Window

The Lightweight Timers window shows lightweight timers, if your application includes the Lightweight Timers component.

Displayed Fields

No. - Queue number

LWTimer Queue - Address serving as identifier of the queue that is created.  The application creates a queue structure (as a global or using malloc()) and uses that address to identify which queue it is operating on

Valid - indicates whether or not the timer queue appears valid. If No, then the data structure is corrupt.

Number - the number of timers in this queue. A given period queue can have a number of timers with different offsets.

Timer Period - The period of each timer in MQX ticks (all lightweight components operate on ticks only)

Time to Fire - The time in ticks before the timers fire

Timer-Detail Fields

LWTimer Address - the address of the timer

Valid - indicates whether or not the timer descriptor appears valid. If No, then the data structure is corrupt.

Timer Offset - Number of ticks to offset from the period.  Every all timers in the queue fire every period ticks (as defined by the queue), unless there is an offset, at which point firing is delayed by offset ticks.

Timer Function - The name and/or address of the function to be called each time the timer is fired

Function Parameter - The parameter that is passed to the Timer Function.
