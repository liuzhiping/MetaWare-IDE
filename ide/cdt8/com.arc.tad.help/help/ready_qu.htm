Ready Queue Window

The Ready Queue window shows all tasks in all ready queues. Tasks are listed in the order in which they appear in the ready queue.

If a task blocks (for example, if it calls _msgq_receive() and no messages are pending), MQX makes active the second task in the ready queue for the priority. If no second task exists, MQX makes active the first task at the next-lower priority.

If a task calls _sched_yield() or if its time slice expires, MQX places the task at the end of the ready queue for its priority and makes active the second task in the same ready queue. If no other tasks are in the same ready queue, MQX makes the task active again.

Displayed Fields

Each entry displays this information for the task:

  • Priority-the priority of the task.
  • Task Identification-the task.
