Identifying Tasks

When MQX creates an instance of a task, it assigns the instance two unique numbers:

  • task ID-a combination of the processor number and an index
  • task descriptor (TD) pointer-the address of the context information for the task

All instances of a particular task have the following information associated with them:

  • task name-supplied by the application in the task template, which MQX uses to create the task
  • function name-the symbol that is associated with the function that represents the task
