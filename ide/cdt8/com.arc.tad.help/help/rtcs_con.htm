RTCS Configuration Parameters Window

The RTCS Configuration Parameters window displays the RTCS configuration.

Displayed Fields

RTCS Version

Version number of RTCS.

RTCS Tasks - Priority

Task priority of the RTCS tasks. This is the value of the global variable _RTCSTASK_priority.

RTCS Tasks - Stack Size

Additional stack size required by RTCS tasks for callback functions. This is the value of the global variable _RTCSTASK_stacksize.

Maximum Sockets

Maximum number of sockets available to RTCS. This is the value of the global variable _RTCSSOCK_max.

Maximum PCBs

Maximum number of outstanding packet control blocks (PCBs) that are available to RTCS. This is the value of the global variable _RTCSPCB_max.

RTCS Queue Base

Value of the global variable _RTCSQUEUE_base. RTCS needs some message queues for its own use. RTCS uses message queue numbers starting at _RTCSQUEUE_base.

UDP Max Queue Size

Maximum number of incoming UDP packets that can be enqueued. It is the value of the global variable _UDP_max_queue_size.

TCPIP Msg Pool

Handle of a pool of messages used to pass commands and data to the RTCS TCP/IP Task.

TCPIP Queue

Queue ID of the input queue for the RTCS TCP/IP Task.

Displayed Fields-RTCS Options

IP Forwarding

Indicates whether IP forwarding is enabled. IP forwarding is the ability of the stack to route packets between network interfaces.

RTO Minimum

Retransmission timeout-minimum value in milliseconds. RTCS dynamically determines the retransmission timeout, but the minimum possible value is displayed as the RTO Minimum.

Bypass Rx Checksum

If the field is True, RTCS does not verify checksums on incoming packets. This means that RTCS does not detect errors in the incoming data stream. However, the probability of errors is low, because the underlying layer includes a checksum that detects errors in the data stream.

Bypass Tx Checksum

If the field is True, RTCS does not generate checksums for outgoing packets.

Displayed Fields-PCB Partition

Handle of the PCB partition, which is the partition from which RTCS allocates PCBs.

Displayed Fields-Socket Partition

Handle of the socket partition, which is the partition from which RTCS allocates socket structures that maintain socket-specific information for each socket.

Controls

TCPIP Msg Pool

Double-click the icon to open the Examine Message Pool window for the TCP/IP message pool.

TCPIP Queue

Double-click the icon to open the Examine Message Queue window for the TCP/IP queue specified.

PCB Partition

Double-click the icon to open the Examine Partition window for the partition that RTCS created to handle PCBs.

Socket Partition

Double-click the icon to open the Examine Partition window for the partition that RTCS created manage socket information.
