Log Summary Window

MQX can log all MQX and RTCS function calls, context switches, and interrupts. You can choose to log all information or a subset of the information in the kernel log. User-defined logs may also be listed.

MQX has logs and LWLogs. The kernel log (klog) uses LWLog #0.The Log Summary window displays all types of logs, and only those created are shown.

Displayed Fields

Log Number

The ID number of the log. This value is a number between zero and 15, inclusive. An ID of 0 denotes the Kernel log.

Current Status

If the corresponding log has not been created by the application, this entry shows Not Created. Otherwise, the current size and maximum size of the log are shown, in words.

When an application creates the log, it sets the maximum size, which is the maximum amount of data (in long words) that can be written into the log. Current log size is the amount of data that has been written into the log so far.

Enabled?

At any time, the application can disable a log (and prevent all write operations) by calling _log_disable(). The application can subsequently enable a log by calling _log_enable().

Overwrite?

  • Yes: When the log becomes full, newest entries overwrite the oldest entries.
  • No: when the log becomes full, no further writing occurs.

Viewing Log Entries

You can view log entries by expanding a Log that is created and has non-zero words used:

Each log entry is represented by an entry in the list. Each entry contains a sequence number, a timestamp, and application-defined data. The log entries are displayed in chronological order, with the oldest entry first. If the first entry is not sequence-number 1, the oldest entries in the log are being overwritten. MQX assigns the timestamp from the kernel time when it logs the entry.

Because MQX defines the format of the data in the kernel log (ID 0), it can display it more effectively than it can display application-defined logs.

The Kernel Log

To view the kernel log, be sure to compile MQX with the MQX_KERNEL_LOGGING compile-time option set to 1. The type of entry determines the information that is stored with the entry, as follows.

Type of Entry
Data
Context switch
  • entry number
  • timestamp
  • Context switch to
  • name of the task
  • task ID
Interrupt generated
  • entry number
  • timestamp
  • interrupt generated
  • interrupt number
Interrupt return
  • entry number
  • timestamp
  • ISR returns

  • entry number
  • timestamp
  • function being entered
  • input parameters
MQX or RTCS function exit
  • entry number
  • timestamp
