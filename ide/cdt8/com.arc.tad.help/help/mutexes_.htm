Mutexes Window

The Mutexes window shows list of mutexes. Each entry contains the current state of the mutex, the attributes of the mutex, the task that has the mutex locked, and a list of the tasks waiting to lock the mutex.

Displayed Fields

Mutex

The name of the variable containing the Mutex descriptor, if it is static. Otherwise, the address of the descriptor.

Locked By

The name of the task, if any, that has this mutex locked. Unlocked is displayed if the mutex is not locked.

Waiting

The number of tasks waiting for this mutex.

Destroy

Indicates if the mutex is in a delayed destroy state.

Boost Count

If the mutex is locked, this field indicates the number of times that the priority of the locking task was boosted. Boost Count has no meaning if the mutex is unlocked.

Displayed Fields-Policy

Waiting Policy

One of the attributes associated with a mutex is the waiting policy that is in effect for tasks that want to gain access to the mutex. The waiting policy is one of the following:

  • Limited Spin
  • Priority Queued
  • Queued-if the mutex is not available, the waiting task is blocked and placed in a waiting queue
  • Spin

Spin Count

Number of _sched_yield() loops that can occur before the mutex-lock request fails. Spin Count is used only if the waiting policy is Limited Spin.

Priority Policy

One of the attributes associated with a mutex is the priority policy. The priority policy may cause the priority of the task that owns the mutex to change while the task owns the mutex.

The priority policy is either priority Inherit or priority protect.

Priority

Minimum priority that an owner task can have while it owns the mutex. If an owner task has a lower priority, its priority is raised to this level. Priority is only used if the priority policy is priority protect.

Displayed Fields-Task List

If the mutex is locked, the locking task is shown.

The waiting-tasks list contains an entry for each task that is waiting for the mutex. The entries contain the task name and task ID of the tasks.
