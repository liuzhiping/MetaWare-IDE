All MQX Data Is Invalid Until Kernel Is Initialized

The debugger informed TAD that the target application was reset. The memory was not changed; therefore, all MQX data from the previous debugging session are still in memory and appear to be valid. The message is a warning that the data being viewed are old data until MQX starts running again and reinitializes the data.
