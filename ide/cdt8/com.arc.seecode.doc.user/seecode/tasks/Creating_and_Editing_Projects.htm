Creating and editing projects

Setting up multiple debugger projects allows you to save different settings and window layouts. Debug projects save you the time required to set options and toggles each time you run a debug session. Debugger Options and Target-Specific Options will be saved to the current project you are debugging.

All debug project information is stored in install_directory/hc<target>/sc/projects/projectname where install_directory is the directory of your full toolset install and hc<target> is the name of your hc directory (hcarc, hcac, hcarm, hcppc).

Note: You can not switch projects while debugging without closing the debugger and restarting. However, you can copy the current project, restart the debugger, and select the project you saved, see Duplicating a Project.

Starting a new project

A new project will provide the default window layout setting and no options.

  1. Launch the debugger.
  2. In the Debug a process or processes window, click Edit Projects.
  3. In the Create/Edit Projects dialog, enter a name for the project. Project names cannot contain spaces.
  4. Click Create. The project will appear in the project list on the right.
  5. Click OK. The new project will be set as the current project.

Duplicating a project

When you duplicate a project, the project window layout and program set options are copied to the new project.

  1. Launch the debugger.
  2. In the Debug a process or processes window, click Edit Projects.
  3. In the Create/Edit Projects dialog, select a project from the list on the right.
  4. In the Destination project name: field enter a name for the new project.
  5. Click Copy. The project will appear in the project list on the right.
  6. Click OK. The new project will be set to the current project.

Removing a project

When you remove a project, all window layout files and program settings (including CMPD options) in that project directory are deleted.

  1. Launch the debugger.
  2. In the Debug a process or processes window, click Edit Projects.
  3. In the Create/Edit Projects dialog, select a project from the list on the right.
  4. Click Delete.
  5. Click OK.

Changing the current project

  1. Launch the debugger.
  2. In the Debug a process or processes window select a project from the Choose project: list.
  3. Click OK to start the debug session.
