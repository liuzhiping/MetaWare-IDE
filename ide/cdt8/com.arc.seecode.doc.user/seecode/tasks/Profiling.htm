Profiling your program

You can view a statistical profile of your application in any of the following ways:

  • using the Profiling window
  • as a separate column in selected windows
  • using profiling-specific windows

Enabling profiling in a debug session

  1. Compile and link your application with profiling information using options -p or -pg.
  2. Start the debugger normally. The debugger automatically detects profiling information in your executable and constructs menus and display windows according to your target and level of compiled profiling information.
  • Tip: For ARC targets, you can enable profiling for instruction counts and killed instructions by setting options using the Target-Specific Options dialog for ARC targets.

Disabling profiling

  1. Select Tools | Debugger Options.
  2. Click the Command-Line Options tab.
  3. Deselect Enable profiling window.

    You can also disable profiling by using the -noprofile option when invoking the debugger on the command-line or adding it to your .args file. For more information on editing the .sc.args file, see the MetaWare Debugger User's Guide.

Viewing columns of profiling data or profiling specific windows

  1. Open the window for which you wish to view profiling information. Displays that commonly include profiling information are: Source, Disassembly, Global Functions, Memory, and Hardware Stack. Inclusion of profiling information in different windows depends on your target.
  2. Right-click in the window.
  3. Select Profiling | <menuitem>, where <menuitem> is any of the available menu items. Menu items on the top of the menu add columns of profiling information such as interrupt counts on the left of the window. Menu items on the bottom open profiling-specific windows.

Troubleshooting profiling

Problem:

Profiling menu item does not appear in Displays menu or Profiling menu item is not available in the right-click menu of any window.

Possible solutions:

  • Recompile your program with profiling information using the -p or -pg options. Also compile with debug information using the -g option.
  • Restart the debugger on the command line using option -profile. Profiling information is enabled by default, but option -noprofile may have been previously set for the debug project.
  • The problem may be a result of your target not supporting profiling for a hardware target or DLL. The ISS fully supports profiling. Try changing your target to the ISS; see Selecting your Target.
