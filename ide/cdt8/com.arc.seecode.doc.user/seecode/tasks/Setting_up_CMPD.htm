Setting up a multi-process debug session

The CMPD Options dialog allows you to add processes to a table one at a time. You can add, edit, or remove processes. To edit or remove a process, select the process in the table and click Edit or Remove.

Creating a new CMPD session

  1. Launch the debugger.
  2. (Optional) Create a new project. This step is not mandatory, but is highly recommended to keep the CMPD settings separate from any single-process debug sessions you have. See Creating and Editing Projects for information on creating a new project.
  3. Click Add Process.
  4. In the Add process dialog enter a name for the process. The process name should start with an alpha character and can not contain any spaces.
    Note: Once the process is created you can not edit the name.
  5. Enter a process number or set of numbers. You can enter one number, an integer, to tell the debugger what number applies to the process. This number will coincide to the target processor. You can also enter a process set, which consists of two integers separated by a colon ':' character. A process set spans a number of processors from the lefthand number to the right. For example, process set 1:3 uses the same program for 3 processes 1 through 3; The next process set might be 4:5.
  6. Enter the full path of the program in the Command-line field or click the Browse button to locate your executable.
  7. Click the Debugger options button to set command-line options and program arguments for this process.
  8. Click the Target-specific options button to set the connection to your target system (hardware or simulator) for this process.
  9. Click OK to add the process.
  10. Repeat steps 3 through 9 until all of your processes are created.
  11. Click OK to save the CMPD settings.
  12. Click OK to start the debug session.
