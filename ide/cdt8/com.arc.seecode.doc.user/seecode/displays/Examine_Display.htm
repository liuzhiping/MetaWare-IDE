Examine window

Use the Examine window to examine program data in type-aware fashion. You can dereference (follow) pointers and traverse into structures, arrays, and unions.

Note:  The Examine window is available only for programs that have been compiled with debugging information (the -g option).

In the Examine window, information about structures, arrays, and pointers can be expanded or collapsed through a tree control. Double-click an item in the tree control to examine related information in a detailed view. For example, if the item represents a function pointer, the debugger opens a source or assembly window showing you that function.

Saving your data to a file

To save your data to a file, right click and select Save to File.

Using deep and shallow bindings

The Examine window can use either "deep" or "shallow" binding for any expression that you enter. The default is shallow binding. Click the Deep button to switch from shallow to deep binding. Click the button again to switch back to shallow binding.

When deep binding is selected, expressions are evaluated as they are first entered; they are not evaluated again. For example, if you enter A[i], i and A are evaluated once. If i = 5, the window forever shows A[5]. This is useful when you want to know who modifies A[5] in the future, even if i changes.

When shallow binding is selected, expressions are re-evaluated each time the window is updated. For example, when the expression is A[i] and the program is in a loop with control variable i, the value changes each time i changes. The expression can become invalid when control leaves the loop (for example, if i is local). The expression can later denote an entirely different array A and integer i.
