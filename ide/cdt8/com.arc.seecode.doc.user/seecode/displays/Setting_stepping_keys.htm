Setting keyboard shortcuts for stepping actions

The Debugger Preferences dialog allows you to modify the keyboard shortcuts, or key bindings, for the standard debugger stepping actions. This feature allows you to optionally use Visual Studio style debugger key layout, or define your own custom keyboard shortcuts. Key combinations for stepping actions involve a command key (or combination of) CTRL, SHIFT, or ALT and one function key (F4-F12).

The stepping toolbar buttons directly reflect changes you make to the stepping keyboard shortcuts. Rolling the mouse cursor over a stepping toolbar button reveals the keyboard shortcut for that action.

The Debugger also has a GUI for setting keyboard shortcuts for user-defined macros and debugger commands, see Setting Keyboard Shortcuts for Commands and Macros.

Opening the stepping shortcut editor

To access the GUI for modifying stepping action keyboard shortcuts:

  1. With the debugger running, select Tools | Debugger Preferences.

  2. Click the Stepping Keys tab.

Using Visual Studio debugger keyboard shortcuts

To use an alternate key layout that emulates the Visual Studio debugger:

  1. Open the Stepping keys editor.
  2. Under the Stepping Shortcut Key Style list, select Visual Studio Debugger.
  3. Click OK or Apply to commit changes.

Setting custom keyboard shortcuts for stepping actions

Before creating your own custom keyboard shortcuts, you can select Debugger Default, or Visual Studio Debugger as a base key map layout from which to begin.
  1. Open the Stepping keys editor.
  2. Under the Stepping Shortcut Key Style list, select Custom.

  3. Select a Stepping action from the table.
  4. (Optional) Select a different function key from the function key list.

  5. (Optional) Select Enable Modifier and select a command key CTRL, SHIFT, ALT, or combination from the command key modifier list.
  6. (Optional) Deselect Enable Modifier, if checked, to remove any command keys added to the keyboard shortcut.
  7. Click OK or Apply to commit changes.
NOTE: No two keyboard shortcuts can share the same key sequence. Distinct values must be set for all keyboard shortcuts in order for the setting to be saved.
