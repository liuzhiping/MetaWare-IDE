Instruction Count History window

The Instruction Count History window shows disassembled code for your program. Labels of the form .0, .1, .2, and so on, are displayed in the left margin. Each label corresponds to a symbolic label and may be used to represent a symbol. This convention saves horizontal space in the window.

These labels change their associations when the Instruction Count History window is updated. You can use a label in a simple expression (for example, to specify the address of a breakpoint) for as long as the same values are on display -- once the Instruction Count History window is closed, all associations are discarded.

When the left margin shows:

  • a green arrow, the program has stopped on the line
  • a magenta arrow, a stack frame is not currently active; the arrow shows where the program will be when control re-enters the stack frame
  • a solid red rectangle, a breakpoint is set on the line; when the letter "T" is displayed, the breakpoint is specific to a thread, which you can identify through the Breakpoints window
  • a grey rectangle, the breakpoint is disabled
