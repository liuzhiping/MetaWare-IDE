Image window

The Image window displays an image from data memory and other sources such as registers (in the case of some processors such as Videocore). See also Image Memory window.

The image may be in one of several formats:

  • a sequence of bytes representing an 8-bit grayscale image
  • a sequence of halfwords representing a 16-bit grayscale image
  • a sequence of 32-bit words representing an ARGB (alpha-red-green-blue) image
  • a sequence of halfwords representing packed RGB (R5, G6, B5)
  • 4:2:0 YUV color
    Note: For an article explaining YUV color, see http://www.joemaller.com/fcp/fxscript_yuv_color.shtml

Configuring the image window

To specify the image display:

  1. Open the Image window by selecting Display | Image from the main menu.
  2. Right-click in the Image window and select Configure | Configure image parameters.
  3. Specify parameters in the Configure image parameters dialog.

Reloading image parameters:

Once having specified the parameters and saved them, you can reload them:

  1. Open the Image window by selecting Display | Image from the main menu.
  2. Right-click in the Image window and select Configure | Reload prior parameters. Note: The Image configuration applies to all Image windows.

Updating the image window

Using the Configure image parameters dialog you can specify whether the display updates the image every time execution stops. If you choose not to do this, such as to save memory traffic, you can click on the Update button to update the window at any time.

Adjusting the image display

An 8-bit grayscale image can be automatically "scaled" -- i.e., the range of values found in the image can be expanded to fill the entire 0..255 range. The 16-bit grayscale image is already scaled to 0..255 (Java does not permit displaying a 16-bit grayscale image, so the debugger converts the values to a 0..255 range).

The image can be flipped and rotated and zoomed by clicking the Flip and Rotate buttons.

Warning: Zooming might not work on Windows NT 4.0; but does work on Windows 2000. This appears to be a bug in Java. the syptom is that the image just disappears for all but 1:1 zoom.

The memory corresponding to the image can be displayed by clicking the Memory button.
