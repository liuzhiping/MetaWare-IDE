Breakpoints window

The Breakpoints window lists all set breakpoints. The Breakpoints window indicates:

  • whether the breakpoint is enabled or disabled
  • the location of the breakpoint in source or disassembly
  • the attributes of the breakpoint, such as count or condition

Use the Breakpoints window to view the status of all set breakpoints. When the left margin shows:

  • a green arrow, the program has stopped on the line
  • a magenta arrow, a stack frame is not currently active; the arrow shows where the program will be when control re-enters the stack frame
  • a solid red mark, a breakpoint is set on the line
  • a gray mark, the breakpoint is disabled

Adding and removing breakpoints

To add a breakpoint:

  1. In the Breakpoints window, click the Set button.
    Or
    Right-click and select Set.
  2. Use the Set Breakpoint dialog to set the breakpoint.

    Note:  You can set counted and/or conditional breakpoints using the Set Breakpoint dialog.

To remove a breakpoint:

  1. In the Breakpoints window, select the breakpoint.
  2. Click the Remove button.
    Or
    Right-click and select Remove.

To remove all breakpoints:

  1. Click the All button beside the Remove button.
    Or
    Right-click and select Remove All.

Enabling and disabling breakpoints

Disabled breakpoints do not halt execution or animation. You can disable a breakpoint or re-enable a disabled breakpoint anytime after the breakpoint has been set.

To disable a breakpoint:

  1. In the Breakpoints window, select the breakpoint.
  2. Click the Enable/disable button.
    Or
    Right-click and select Enable/disable.

To disable all breakpoints:

  1. Click the All button beside the Enable/Disable button.
    Or
    Right-click and select Enable/Disable All.
