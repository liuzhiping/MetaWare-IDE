Starting the debug session

Use the Debug a process or processes window to start your debug session.

  1. (Optional) Select Use debug project and select or create a project for your debug session. To create a new project, see Creating and Editing Projects. If you leave Use debug project deselected the debugger will create a directory named .sc.project in the directory where the debugger was launched. All debugger configuration files will be placed in the .sc.project directory.
  2. Select the type of debug session:
    • Create new process (single process session) will begin a single process session that involves one program running on one target.
    • Multi-process session will begin a Coordinated Multi Process Debug session (CMPD). Select Multi-process session when you are debugging multiple programs or multiple targets. If you are starting a CMPD session, see Setting up a Multi-Process Debug Session.
  3. Enter the full path of the program in the Program & args field or click the Browse button to locate your executable. You can provide program arguments and additional debugger command-line arguments following the path.
  4. Click the Debugger options button to set debugger options for a single process session. If you select Multi-process session, click the CMPD Options button.
  5. Click the Target-specific options button to set the connection to your target system (hardware or simulator).
  6. Click OK or press the [ENTER] key to start the debug session.
