Stepping your program

Use the stepping toolbar to execute your program. The debugger allows you to step through machine instructions (Disassembly) and source statements simultaneously. The table below shows each button and its function.

Icon

Name

Function
Restart Restart the process with its original parameters without losing previously set breakpoints.
Run Runs the process at full speed; execution stops only at breakpoints and hardware watchpoints, or if you explicitly stop it using the Stop button or command.
Stop Halts execution of a running process.
Src Into Steps one source statement at a time, stepping into function calls.
Src Over Steps one source statement at a time, stepping over function calls.
Instr Into Steps one machine instruction at a time, stepping into function calls.
Instr Over Steps one machine instruction at a time, stepping over function calls.
Step Out Runs until the end of the current function and return to the instruction or source-code statement following the function call.
Animate Animate the process; execute process in slow motion: one source-code statement or one machine instruction at a time. Animation is similar to executing multiple step commands. You can also animate commands. See also Animation.
+ / - Increase or decrease the speed of animation.
