Setting keyboard shortcuts for commands and macros

The Debugger Preferences dialog allows you to create, change, and remove keyboard shortcuts, or key bindings, for debugger commands and user-defined macros. This feature allows you to easily access macros and debugger commands by entering a keyboard sequence involving a command key or combination of CTRL, ALT, and SHIFT plus a numeric key (0-9).

The Debugger also has a GUI for setting keyboard-shortcuts for the stepping actions, see Setting Keyboard Shortcuts for stepping commands.

For a list and descriptions of available debugger commands, see the Debugger Command Reference. Or type help all in the Command field.

Opening the keyboard shortcut editor

To access the GUI for adding, changing, and removing keyboard shortcuts for commands and macros:

  1. With the debugger running, select Tools | Debugger Preferences.

  2. Click the Command/Macro Keys tab.

Adding a keyboard shortcut for a user-defined macro

  1. Define or load your macro definitions into the debugger (see the MetaWare Debugger User's Guide for more information).

  2. Open the Command/Macro keys editor.

  3. The macros you loaded into the debugger will appear in the table. Select a macro from the table.

  4. (Optional) You can modify the macro command using the text field.
    Note: If you modify the command string of the macro, the originally defined macro command will still appear in the table when it is reloaded. Click the Reload Defined button to synchronize the defined values with the table.

  5. Select a command key or command key combo from the first drop-down list.

  6. Select a number key from the second drop-down list.

  7. Click Apply or OK to commit changes.

Adding a keyboard shortcut for a command

See the Command reference for a list of commands you can use.

  1. Open the Command/Macro keys editor.

  2. Click Add.

  3. Type the name of a command in the text field.

  4. Select a command key or command key combo from the first drop-down list.

  5. Select a number key from the second drop-down list.

  6. Click Apply or OK to commit changes.

Editing a keyboard shortcut

  1. Open the Command/Macro keys editor.

  2. Select an entry from the table.

  3. (Optional) Edit the name of a command or macro in the text field.

  4. (Optional) Select a command key or command key combo from the first drop-down list.

  5. (Optional) Select a number key from the second drop-down list.

Removing a keyboard shortcut

  1. Open the Command/Macro keys editor.

  2. Select an entry from the table.

  3. Click Remove.

Help on setting keyboard shortcuts

The following are a few guidelines for creating keyboard shortcuts:

  • You must specify both a command key, or combination such as CTRL+SHIFT, and a number key (0-9) for every keyboard shortcut. A value for both command and number keys must be specified in order for the setting to be saved.

  • No two keyboard shortcuts can share the same key sequence. Distinct values must be set for all keyboard shortcuts in order for the setting to be saved.

  • Clicking the Reload Defined button will populate the table with ONLY the commands and macros that have already been defined. If you edited the command string of a macro the table will add the original macro command string as it was received from the debugger.

  • If you add a new keyboard shortcut and do not specify a name other than the default "<undefined>," the changes will be lost. The GUI ignores entries named "<undefined>," and will not keep these rows in the table after it the Debugger Preferences dialog has been closed.
