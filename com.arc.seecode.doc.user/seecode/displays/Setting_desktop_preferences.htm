Setting desktop preferences

Use the options in the Debugger Preferences dialog and Desktop tab to set the type of desktop environment you prefer. Three distinct styles of desktop environment are provided for you: MDI, Sash-panel, and SDI. Sash-panel is enabled by default.

To change your debugger desktop environment, select from the Desktop Model list as in the following figure.

The table below describes the desktop model types and their properties.
Desktop ModelDescription Available Options
MDI Style Displays appear as internal windows to the debugger desktop window. Internal windows can be minimized, maximized, and restored. You can move a display to another internal window by dragging and dropping the tab to a new location.

Menu items for minimizing, restoring, cascading, and tiling are added to the Windows menu. Windows can also be "docked" to stick in a position. Docked windows cannot be minimized.

  • Save window layout on exit
  • Show close buttons on tabs
  • Snap to grid
  • Set docking regions enabled (top, bottom, left, and right)
Sash-panel Style Displays appear as panels inside the debugger main window. You can resize a panel by dragging its divider. You can move a panel to another location by dragging and dropping the title bar or tab to a new location.
  • Save window layout on exit
  • Show close buttons on tabs
  • One-touch-expandable divider
SDI Style Displays appear as individual windows on the desktop of the operating system. You can move a display to another window by dragging and dropping the tab to a new location.
  • Save window layout on exit
  • Show close buttons on tabs

Save Window Layout on Exit: (All desktop models) If checked will automatically save the arrangement and placement of displays when you exit the debugger.

Show close buttons on tabs: (All desktop models) If checked will place a button on the left side of every tab. Roll over the tab to show the button, and click the button to close the panel represented by the tab such as the following example:

Special feature for Sash-panel desktop style: If Show close buttons on tabs is selected, the close   button on the panel will close all contained displays.

Snap to grid Windows: (MDI only) If checked the internal windows will automatically snap to an invisible grid.

Docking regions enabled: (MDI only) For each region checked, you can drag a window to that edge of the debugger desktop screen and the window will "stick" there. Docked windows cannot be minimized and will resize proportionally to the debugger desktop as its size changes.

  • Top -- Drag the window to the top of the debugger frame
  • Bottom -- Drag the window to the bottom of the debugger frame
  • Left -- Drag the window to the far left of the debugger frame
  • Right -- Drag the window to the far right of the debugger frame

  • To remove the window from the dock, click the title bar and drag it away from the edge where it is docked.

One-touch-divider: (Sash-panel only) If checked will show buttons on each panel divider that allow you to minimize and maximize the panel.
