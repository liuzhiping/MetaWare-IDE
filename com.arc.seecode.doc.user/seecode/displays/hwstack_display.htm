Hardware Stack window

The Hardware Stack window is a variant of the Memory window. The Hardware Stack window shows memory with the display width being the natural size of elements on the runtime stack. The stack pointer register is displayed to the right of the line whose address equals the value of the stack pointer.

The stack can be viewed in ascending or descending order of memory. To reverse the order, click the memup button.

For stacks that grow down, memory is displayed in descending addresses, and the stack pointer register stays near the bottom. For stacks that grow up, memory is displayed in ascending order, and the stack pointer register stays near the top. The displayed addresses change to ensure that the stack pointer register is always visible.

To change the base numbering system for displaying memory, choose one of the following options from the Base box:

  • 16 (hexadecimal)
  • 10 (decimal)
  • 8 (octal)
  • 2 (binary)

Modifying the value of a memory address

To modify the value of a memory address:

  1. Click the value that you want to change, and then click the Change button.
    Or
    Right-click the value, and then click Change.

    The Change dialog is displayed.

  2. Modify the value in the New value field of the Change dialog, and then click OK.
